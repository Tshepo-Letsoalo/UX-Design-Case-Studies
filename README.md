## üí° The Design Thinking Process

I adhere to a robust, iterative design thinking methodology to ensure all solutions are anchored in user needs and tested thoroughly. 




| Phase | Core Goal | Key Activities |
| :--- | :--- | :--- |
| **Empathize** | Understand user needs and pain points. | User Interviews, Surveys, Competitive Audits, Creating Personas. |
| **Define** | Clearly state the user problem. | Writing Problem Statements, User Stories, and establishing HMW (How Might We) questions. |
| **Ideate** | Brainstorm creative solutions. | Sketching, Crazy Eights, Storyboarding. |
| **Prototype** | Create tangible representations of designs. | Low-fidelity (paper) and High-fidelity (digital, interactive) prototypes (Figma, Adobe XD). |
| **Test** | Evaluate the prototype with users. | Usability Studies, A/B Testing, Synthesizing Test Findings, and Iteration. |
